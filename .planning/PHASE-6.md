### 6B-PRE-03 — Compliance score: hero metric on dashboard

**Status: 🔴 Not started**  
**Priority: High (core product value signal)**

**What to build:**

A single prominent number at the top of the admin dashboard showing: `X% of your team is currently certified`. Computed from existing data — no new DB queries needed (data is already fetched in the page).

**Logic:** `certified / totalMembers * 100`, where `certified` = members with a cert whose `expires_at > now`.

**Implementation:**

1. In `app/dashboard/page.tsx`, after `memberDetails` is assembled, compute:
   ```ts
   const certifiedCount = memberDetails.filter(m => m.trainingStatus === 'passed').length
   const totalCount = memberDetails.length
   const complianceScore = totalCount > 0 ? Math.round((certifiedCount / totalCount) * 100) : 0
   ```
2. Pass `complianceScore`, `certifiedCount`, `totalCount` as props to a new `ComplianceScore` component
3. Place the component in the admin view between the firm header and the invite section
4. Component renders: large number (`text-4xl font-bold`), label ("of your team is certified"), and a sub-line (`certifiedCount of totalCount employees`)
5. Color the number: `text-teal-400` when 100%, `text-amber-400` when 50–99%, `text-red-400` when <50%
6. Edge case: 0 members → show "—" instead of "0%"

**Files to touch:**
- `app/dashboard/page.tsx` — add computation + pass props
- `app/dashboard/_components/compliance-score.tsx` — new component

### 6B-PRE-06 — Onboarding checklist

**Status: 🔴 Not started**  
**Priority: High (activation flow)**

**What to build:**

A dismissible getting-started checklist shown to a firm admin on their first login after payment. Permanently dismissed once all steps are complete (or manually closed by the admin).

**Three steps:**
1. ✅ Purchase complete
2. ◻ Invite your team members
3. ◻ They complete training → you're compliant

**Persistence:** Store dismissed state in Supabase `firms` table as a `boolean` column, or use a `training_events` row with `event_type = 'onboarding_dismissed'`. The latter requires no schema change.

**Implementation steps:**

1. Add migration `0008_onboarding_dismissed.sql`: add `onboarding_dismissed boolean not null default false` column to `firms` table
2. In `app/dashboard/page.tsx`, fetch `onboarding_dismissed` as part of the existing firms query
3. Compute checklist step completion from existing data:
   - Step 1: always complete (they're in the dashboard)
   - Step 2: `members.length > 0` (excluding the admin themselves)
   - Step 3: `memberDetails.some(m => m.trainingStatus === 'passed')`
4. Show checklist if `!firm.onboarding_dismissed`
5. "Dismiss" button → POST to a new route `app/api/firm/onboarding/dismiss/route.ts` that sets `onboarding_dismissed = true`
6. On all steps complete, auto-dismiss with a "You're compliant!" success state (show briefly, then disappear)
7. Place checklist between the compliance score and the invite section

**Files to touch:**
- `supabase/migrations/0008_onboarding_dismissed.sql` — new column on firms
- `app/dashboard/page.tsx` — add fetch + conditional rendering
- `app/dashboard/_components/onboarding-checklist.tsx` — new component
- `app/api/firm/onboarding/dismiss/route.ts` — new route (POST, sets flag)
